hexo + github pages 搭建 个人博客

- hexo 是什么么？
- Hexo是一个快速，简单和强大的博客框架。您在Markdown（或其他语言）中撰写帖子，Hexo会在几秒钟内生成具有美丽主题的静    态文件。（hexo 官网）
- Hexo是一个开源的静态博客生成器,用node.js开发,作者是台湾大学生tommy351。(百度百科)
- hexo 就是为 博客 而生 ，将写好的文章保存为 .md 文件（Markdown 文件），hexo就把 .md 格式文件自动生成HTML静态页面
- 对的，就是这么简单
  • node npm/cnpm git 环境配置好 （如下操作基于 node git 环境搭建好，github账号设置好）
    • 安装hexo
      $ npm/cnpm install hexo-cli -g

$ hexo init Blog 这个命令在当前目录下生成一个 Blog 文件夹 和 基本配置

$ cd Blog 切换到Blog目录下

| .git git版本控制的管理信息 （不用理）
| scaffolds ==> scaffolds英文翻译为 支架 、脚手架 哪就是顾名思义 吧 （依旧勿扰）
| source ==> 资源（文章放置地方）
| themes ==> 主题（HTML主题）
| .gitignore ==> git 忽略监听文件
| .gitmodules ==> 大概顾名思义吧 git模块（也不懂）
| ._config.yml ==> 站点配置文件。可以在此配置大多数设置。
| package.json ==> webpack配置文件

以上目录有 package.json 没 node_modules ，就要

$ npm/cnpm install 安装依赖包

$ hexo serve 或者 hexo server 启动服务

  • 在浏览器是输入 localhost:4000

  • 服务器是启动了，但是只有空白页，有个警告：WARN No layout: index.html

  • 警告大概是 index.html 没有布局，打开themes文件 发现是空的 一个主题也没有

  • 导致没有主题是 hexo init Blog 初始化时没有下载主题

  • 自己想了两种解决方案：

    . 一、把 Blog 删了，重新 hexo init Blog 一下

    . 二、下载主题 修改主题设置

  • 考虑到删了，重新 init 还是不行，岂不白忙

  • 既然删不起，就找主题吧 https://hexo.io/themes/

  • 找到一个满意的主题，点击进去，找到 githu标志图 点击进去 就能找到主题的github仓库了

  • $ cd themes 到主题目录下，$ git clone https://github.com/Haojen/hexo-theme-Anisina

  • 发现下载速度比乌龟还慢，想了想，我只是把主题下载下了，没有修改什么配置，直接手动下载

  • 窝草，居然秒下，瞬间就下载好了，我也想不明白，同样一个东西，不同方式下载区别辣么大，无语…

  • 把下载好的hexo-theme-Anisina.zip 解压到 Blog/themes目录下，改名为 Anisina

  • 修改主题配置 打开_config.yml 拉到最下 修改为自己的主题

  • 重启服务 运行 localhost:4000 页面出来 这只是预览

$ hexo generate

  • 这样就是生成一套静态文件 （在 public 文件下）这个就是博客了，把推送到github就能用了

如果运气好，就一个命令完事

` $ hexo init Blog `

  • 因为 node_modules 和 默认 主题都下载好了，启动服务就能运行了

    配置 github 上线

  • 创建github仓库 命名为 onelazycat.github.io (前缀是github账户名，我的github名为onelazycat) 后缀固定 github.io

  • 把仓库和本地Blog关联起来

  • 命令 $ vim _config.yml 进行编辑 （vim编译器编辑）

  • 翻到最下面，改成我这样子的

  • 编辑时 key 和 value 有个空格，颜色变了才成功 （例如： type: git 中间有空格）

  • 执行命令

$ npm install hexo-deployer-git --save

  • 在执行命令

    $ hexo deploy

    • 此时在浏览器输入 https://onelazycat.github.io/

    • 很幸运页面出来了，一次性通过
